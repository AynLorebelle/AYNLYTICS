# Aynlytics — Project overview and changes

This document explains the project layout, key features and the recent updates made to the Aynlytics repository. It replaces and supersedes the older "AYNTRACK" overview.

Summary (high-level)
- Aynlytics is a full-featured personal finance tracker with authentication, role-based admin controls, categories, income/expense CRUD, budgets, dashboard analytics, reports, and search + pagination.
- Recent improvements focused on data-driven dashboards, charts, improved validation/sanitization, soft deletes, authorization policies, performance hardening (eager loading & pagination), and tests.

Where to look (top-level files)
- app/
  - Models: `User`, `Category`, `Expense`, `Income`, `Budget` (with relationships and helpers)
  - Http/Controllers: DashboardController, ExpenseController, IncomeController, CategoryController, BudgetController, ReportController, Admin/AdminController
  - Http/Requests: validation and sanitization (RegisterRequest, ExpenseRequest, IncomeRequest, CategoryRequest, BudgetRequest)
  - Policies: ExpensePolicy, IncomePolicy, BudgetPolicy, CategoryPolicy
  - Exceptions/Handler.php: custom render logic for production-friendly error pages
- database/
  - migrations: tables for categories, expenses, incomes, budgets; plus updates to users and soft-delete migrations
  - seeders: CategorySeeder and DatabaseSeeder updated for system/user categories
- resources/views
  - Layout & shared partials: `layouts/app.blade.php`, `partials/alerts.blade.php`
  - Views for resources: expenses, incomes, categories, budgets, reports, admin users
- routes/web.php: registered resource routes and grouped admin routes protected by Gate/authorization
- tests/: feature tests for authentication, CRUD, and dashboard behaviour were added

Recent dashboard & analytics improvements
- Dashboard now shows fully dynamic data:
  - Top stats are calculated from the database (budgets, monthly income, monthly expenses) for the chosen month/year
  - Percent-change calculations compare current month vs previous month. Safe-handling for division-by-zero is implemented:
    - previous month == 0 and current == 0 → shows "N/A"
    - previous month == 0 and current > 0 → shows "New"
    - otherwise shows percentage (rounded) with green/red coloring for positive/negative changes
  - Savings percentage calculated from income vs expenses
  - Recent Transactions list is now generated by merging incomes and expenses, sorted by transaction_date DESC and limited to the most recent 10 items
- Charts: A server endpoint provides JSON used by client-side Chart.js components (expenses by category, income/expense trends, budget performance, daily spending)

Database & migrations
- Tables and soft-delete support:
  - `users` (role enum `user|admin`, profile fields)
  - `categories` (name, type enum `income|expense`, is_system flag, user_id)
  - `expenses` and `incomes` (user_id, category_id, amount, transaction_date, description, soft deletes)
  - `budgets` (user_id, category_id, amount, month, year, soft deletes)

Security, validation & authorization
- FormRequests sanitize input (trimming, numeric cleanup) and provide friendly messages.
- Policies & middleware protect resource access; `can:admin` gate used for admin routes.

Testing & CI
- Feature tests cover core auth and CRUD paths and the dashboard features described above.
- Tests run using sqlite memory for speed: `php artisan test`.

How to run locally (quick)
1. Copy `.env.example` to `.env`, set DB credentials and run `php artisan key:generate` if needed.
2. Install dependencies and migrate:
   - composer install
   - php artisan migrate --seed
   - php artisan serve
3. Visit `http://127.0.0.1:8000` and register a user.

Notes & follow-ups
- The dashboard is prepared for future improvements like cached analytics, drill-down / modal views for chart items, and a Trash/Restore UI for soft-deleted records.
